> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the X Monitor app — cookie-authed thread saving, full thread reconstruction, engagement tracking, author watches with filters, yt-dlp fallback for one-off saves.

**Tech Stack:** Python, httpx, yt-dlp (subprocess), FastAPI, React, TypeScript, Tailwind, Vitest, pytest-asyncio

## Task 1: XFetcher Backend + Tests

**Files:**
- Create: `tinyagentos/knowledge_fetchers/x.py`
- Create: `tests/test_knowledge_fetcher_x.py`

- [ ] **Step 1: Write failing tests**

Test `fetch_tweet` and `fetch_tweet_ytdlp` with mocked responses. Test `reconstruct_thread` with a chain of mock tweets. Test `fetch_author_timeline` with mocked GraphQL response.

- [ ] **Step 2: Implement XFetcher**

Key functions:

**Cookie-auth path (GraphQL):**
- `fetch_tweet(tweet_id, cookies, http_client) -> dict` — single tweet via GraphQL TweetResultByRestId
- `reconstruct_thread(tweet_id, cookies, http_client) -> list[dict]` — walk reply chain up to root, down through author's replies, stitch chronologically. Other replies stored separately.
- `fetch_author_timeline(handle, cookies, http_client, count=20) -> list[dict]` — UserTweets GraphQL
- `fetch_bookmarks(cookies, http_client) -> list[dict]` — Bookmarks GraphQL

Headers: `Cookie: auth_token={token}; ct0={ct0}`, `x-csrf-token: {ct0}`, `authorization: Bearer AAAAAAAAAAAAAAAAAAAAANRILgAAAAAAnNwIzUejRCOuH5E6I8xnZz4puTs=...`

**yt-dlp fallback:**
- `fetch_tweet_ytdlp(url) -> dict` — `yt-dlp --dump-json --no-download {url}`, parse for tweet text, author, media, engagement

**Cookie retrieval:**
- `get_x_cookies(agent_browsers_manager) -> dict | None` — calls Agent Browsers API to get auth_token + ct0 from any profile with X login status

- [ ] **Step 3: Wire into IngestPipeline**

Add `if source_type == "x"` branch — try cookie auth first, fall back to yt-dlp.

## Task 2: X API Routes + Author Watch CRUD

**Files:**
- Create: `tinyagentos/routes/x.py`
- Create: `tests/test_routes_x.py`
- Modify: `tinyagentos/app.py`

Endpoints:
- `GET /api/x/tweet/{tweet_id}` — fetch single tweet
- `GET /api/x/thread/{tweet_id}` — fetch reconstructed thread
- `GET /api/x/author/{handle}` — author's recent timeline
- `GET /api/x/bookmarks` — user's bookmarks (cookie auth required)
- `GET /api/x/auth/status` — cookie auth status
- `POST /api/x/watch` — create author watch rule (handle, filters, frequency)
- `GET /api/x/watches` — list all watch rules
- `PUT /api/x/watch/{handle}` — update watch rule
- `DELETE /api/x/watch/{handle}` — delete watch rule

Author watches stored in SQLite table `x_author_watches` with columns: handle, filters (JSON), frequency, enabled, last_check, created_at. The MonitorService integration polls watched authors on schedule and ingests matching posts.

## Task 4: XMonitorApp Component + Registration

**Files:**
- Create: `desktop/src/apps/XMonitorApp.tsx`
- Modify: `desktop/src/registry/app-registry.ts`

Two-state layout (list/detail).

**List view:** Sidebar (Watched Authors with post counts, Saved Threads, Bookmarks, Monitored, Categories, Status). Cards: @handle + avatar placeholder, tweet text (2-line clamp), engagement stats, media badge, monitoring badge.

**Detail view:**
- Reconstructed thread (all author tweets stitched, quote tweets inline)
- Reply tree below (collapsible, capped at 3 levels)
- Engagement stats bar with trend indicators
- Tabs: Thread, History (engagement graphs over time), Metadata
- Action bar: Open on X, Re-fetch, Monitor, Stop monitoring, Delete

**Author watch management:** In sidebar under "Watched Authors" — "+ Watch Author" button opens form (handle input, filter toggles: all posts/min likes/threads only/media only, frequency picker). Click existing watch to edit.

**Auth indicator:** Sidebar shows cookie auth status (green/red dot). "Log in via Agent Browser" link if not authenticated.

Registration: `id: "x-monitor"`, `icon: "at-sign"`, `launchpadOrder: 15.5`
