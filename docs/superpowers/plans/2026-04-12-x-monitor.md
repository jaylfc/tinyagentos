# X Monitor Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the X Monitor app — cookie-authenticated X content capture with thread saving, reply-chain monitoring, engagement tracking, and author watch rules — writing all captured content into the shared Knowledge Base Service.

**Architecture:** `XMonitorApp.tsx` is a platform desktop app (singleton, `category: "platform"`) registered in `app-registry.ts`. It drives a `x-monitor.ts` TypeScript client that calls the TAOS API. The backend fetcher `tinyagentos/knowledge_fetchers/x.py` handles cookie-authed X API access, thread scraping, and reply-chain assembly. The fetcher is wired into `IngestPipeline` as the handler for `source_type="x"` and into `MonitorService` for polling (30 min base, 2× decay, 14-day idle stop). Auth uses cookies read from the agent browser container Chromium profile via the existing `AgentBrowsersManager`.

**Tech Stack:** React + TypeScript (desktop), Python 3.10+ / FastAPI (backend), httpx with cookie jar, existing `KnowledgeStore` + `IngestPipeline` + `MonitorService`, `AgentBrowsersManager` for cookie export.
